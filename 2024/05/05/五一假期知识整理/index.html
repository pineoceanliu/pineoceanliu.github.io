知识点：中间不行看两边

好吧，这是我自己编的

这个思想的主要内容就是把要求的数据从它两侧推出来

如下题

给定一个长度为 n 的正整数序列 a 定义一个序列的子串为 $a_i$, ai+1...aj (1 ≤ i ≤ j ≤ n) ，j − i + 1 为这个子串 的长度。 你的任务是：在最多改变序列中的一个值（可以改成任何整数）的条件下， 找到一个最长的严格上升子串 输入格式 第一行一个整数 n 。 接下来一行 n 个正整数，表示序列 a1, a2...an 输出格式 输出一行一个整数，表示最长严格上升子串的长度 样例输入 6 7 2 3 1 5 6

我们想要直接求无从下手，那我们求a_i 改变后的最长上升子串就行了。

类似的是，这种思想与容斥原理可以联系记忆

我们要求选四个的方案
$$
选四个的方案数=最多选四个的方案数-最多选三个的方案数
$$
就类似这种思维

中值滤波是个模拟题+找规律，过

注意点：LCA多用倍增思想

求过LCA的树，用f[i][logr] 数组跳

O(n)与O(log n)的差距！！！！

CF1969 ( Div. 2)

A题找规律，过。

B题：

给定一个二进制字符串 s，通过对其进行一系列操作，每个操作是选择一个子串并对其进行循环移位，使得整个字符串变得非递减有序。每次操作的成本是所选择子串的长度。要求计算实现这一目标的最小总成本。
void solve(){
cin>>s;
ll ans=0,len=strlen(s)-1,cnt=0;
for(int i=0;i<=len;i++){
if(s[i]=='0'){
if(cnt)ans+=cnt+1;
}else{
cnt++;
}
}
cout<<ans<<"\n";
} `

思路，从左向右，把所有1带上，把0挪到后面去

不过思路是动态规划

dp[i][j]当前的最小和

dp[i][j] = min(dp[i][j], dp[i - d][j - d] + (d + 1) * x);

知识点汇总

学习了几个知识点

最大匹配相关性质

最大独立集=n-最大匹配

最大匹配=最小点覆盖
